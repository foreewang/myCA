# 克隆挑取补偿误差分析与当前应对方案

## 1. 当前结论

当前补偿链路已经从单次开环补偿，升级为可配置的三层方案：

```text
一次补偿计算
  -> 可选 scale 比例修正
  -> 可选 approach 同向逼近
  -> 可选 closed_loop 复拍识别闭环补偿
```

结合人工标注测试结果，目前判断：

1. 视觉识别、人工标注、补偿计算链路整体方向是正确的。
2. X 轴丝杆反向间隙明显，实测约 `15000 pulse`，需要通过同向逼近降低影响。
3. Y 轴丝杆间隙满足当前要求，可近似忽略。
4. 单点 scale 修正能显著改善一次补偿精度，但换点后仍可能有残差，说明误差不完全是单一线性比例问题。
5. 二次补偿验证有效，说明“补偿后复拍 -> 重新定位 -> 再补偿”的闭环思路是当前最可靠的落地方案。

## 2. 补偿计算链路

当前核心逻辑在 `workflow/compensate_executor.py`。

补偿目标计算流程：

```text
clone.center_px
  -> offset_from_image_center_px
  -> mm_per_pixel
  -> offset_right_mm / offset_down_mm
  -> pulses_per_mm
  -> view_signs
  -> compensate_target
```

当前公式为：

```python
target_x = base_x - x_sign * offset_down_mm * ppm * x_scale
target_y = base_y - y_sign * offset_right_mm * ppm * y_scale
```

注意坐标映射关系：

```text
图像 Y 偏移 -> 位移台 X 轴
图像 X 偏移 -> 位移台 Y 轴
```

因此：

```text
scale.x 修正图像上下方向的补偿量
scale.y 修正图像左右方向的补偿量
```

## 3. 当前误差来源

### 3.1 机械间隙

X 轴丝杆间隙较大。实际测试中：

```text
正向移动 10000 pulse 后，如果反向，需要约 25000 pulse 才能产生等效反向位移
```

因此可推断：

```text
X 轴反向间隙约 15000 pulse
```

当前处理方式：

```json
"approach": {
  "enabled": true,
  "x_direction": 1,
  "y_direction": 1,
  "x_margin_pulse": 30000,
  "y_margin_pulse": 0
}
```

含义：

```text
X 轴先移动到 target_x - 30000
再从固定方向移动到 target_x
```

`x_margin_pulse` 建议大于反向间隙，当前用 `30000` 作为保守值。

### 3.2 比例误差

人工标注测试中出现过：

```text
补偿前：center = [3328, 1794]
目标中心：[2560, 2560]
补偿后：center = [2551, 2763]
```

计算得到：

```text
图像 X 方向基本补准：after_x 偏差约 -9 px
图像 Y 方向过补：after_y 偏差约 +203 px
```

因为图像 Y 方向对应位移台 X 轴，所以对 X 轴补偿量加入比例修正：

```json
"scale": {
  "x": 0.79,
  "y": 1.0
}
```

这说明：

```text
当前 X 轴方向有效补偿量偏大，先按 0.79 缩放
Y 轴方向补偿量基本保留
```

### 3.3 单点 scale 的局限

单个特征点调出来的 `scale` 不一定能覆盖所有区域。换点后误差可能再次出现，原因可能包括：

- 图像边缘畸变。
- 物镜视野非严格线性。
- 平台 X/Y 比例误差不是常数。
- 人工标注中心和视觉算法中心存在差异。
- 不同方向运动时机械状态不同。

因此 `scale` 是当前的工程修正手段，不是最终完整标定模型。

## 4. 当前代码应对方案

### 4.1 scale 比例修正

配置位置：

```json
"compensate": {
  "scale": {
    "x": 0.79,
    "y": 1.0
  }
}
```

代码读取位置：

```text
workflow/run_task.py
  build_pipeline_params()
    -> compensate_scale

workflow/compensate_executor.py
  _calc_compensate_target()
```

结果 JSON 中会输出：

```json
"scale": {
  "x": 0.79,
  "y": 1.0
}
```

### 4.2 approach 同向逼近

配置位置：

```json
"compensate": {
  "approach": {
    "enabled": true,
    "x_direction": 1,
    "y_direction": 1,
    "x_margin_pulse": 30000,
    "y_margin_pulse": 0
  }
}
```

代码位置：

```text
workflow/stage_executor.py
  move_to_absolute_with_approach()
```

执行逻辑：

```text
pre_x = target_x - x_direction * x_margin_pulse
pre_y = target_y - y_direction * y_margin_pulse

先移动到 pre_target
再移动到 compensate_target
```

结果 JSON 中应出现：

```json
"move_result": {
  "approach": {
    "enabled": true,
    "pre_target": {},
    "pre_move": {},
    "final_move": {}
  }
}
```

如果结果中 `approach` 为空 `{}`，说明请求体未传入或服务未重启。

### 4.3 closed_loop 二次视觉闭环

配置位置：

```json
"compensate": {
  "closed_loop": {
    "enabled": true,
    "max_iterations": 2,
    "tolerance_px": 30,
    "save_dir": "C:/colony_system/data/compensate_eval/closed_loop",
    "filename_pattern": "closed_loop_{task_id}_{well}_iter{iteration:02d}.bmp",
    "detect_entrypoint": "vision.vision.detect_pipeline:process_image",
    "selector": {
      "mode": "nearest_image_center"
    }
  }
}
```

代码位置：

```text
workflow/compensate_executor.py
  _run_closed_loop()
```

闭环逻辑：

```text
第一次补偿
  -> 读取 move_result.after 作为当前电机坐标
  -> 拍摄复查图
  -> 调用 vision 识别
  -> 构造内存中的 image_item
  -> 判断最近中心的克隆是否进入 tolerance_px
  -> 如果未满足，自动再次计算并移动
  -> 最多执行 max_iterations 次
```

重要说明：

```text
closed_loop 不需要后端二次请求
closed_loop 不需要人工再生成 manual_detect_result_2.json
closed_loop 会在内存中完成“复拍图 result”的等价结构
```

当前实现不是重新打开串口再读电机坐标，而是使用上一次移动结果中的：

```text
move_result.after.x.current_pos
move_result.after.y.current_pos
```

由于补偿后立即复拍，中间没有其它运动，这个坐标可作为复拍基准。

## 7. 人工二次补偿验证方法

人工闭环验证时，第一次补偿后的复拍图没有完整扫描任务 `result.json`。因此新增工具：

```text
tools/make_manual_detect_result_from_stage.py
```

用途：

```text
用复拍图片 + 人工点击点 + 当前电机坐标
生成新的 manual_detect_result_2.json
```

示例：

```powershell
python tools\make_manual_detect_result_from_stage.py `
  --image C:\colony_system\data\compensate_eval\closed_loop\your_image.bmp `
  --cx 2654 `
  --cy 2605 `
  --stage-x 6769277 `
  --stage-y -1536518 `
  --plate-type 24-well `
  --well B4 `
  --objective 4x `
  --fov-width-mm 3.22 `
  --fov-height-mm 3.22 `
  --clone-id C_MANUAL_002 `
  --image-index 1 `
  --out C:\colony_system\data\compensate_eval\manual_detect_result_2.json
```

人工验证通过后，实际落地使用 `closed_loop.enabled = true` 由视觉算法自动完成该步骤。

## 8. 后续优化方向

当前方案满足工程测试需求，但仍有进一步优化空间：

1. 多点残差标定  
   用中心、左上、右上、左下、右下等多个点拟合更稳定的补偿模型。

2. 分轴 pulses_per_mm  
   如果 X/Y 轴比例长期不同，可将 `pulses_per_mm` 从单值升级为 X/Y 分轴参数。

3. 仿射矩阵补偿  
   如果误差和位置相关，单独 `scale.x/y` 不足，应升级为二维矩阵映射。

4. 实时读取闭环拍照坐标  
   当前使用 `move_result.after` 作为闭环拍照坐标，后续可在复拍前实时读取 X/Y 坐标，提高鲁棒性。

5. 目标跟踪选择器  
   当前 closed_loop 默认使用 `nearest_image_center`，若复拍图中多个克隆靠近中心，后续应根据面积、形状、上一轮目标特征做目标匹配。
