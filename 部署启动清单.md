# 部署同步清单

> 适用场景：新电脑上的项目路径固定为 `D:\colony_system`，不再改回 `C:\colony_system`。

## 0. 日常手动启动服务

每次需要手动启动服务时，打开 **Anaconda Prompt**，按顺序执行：

```bat
conda activate colony_system
cd /d D:\colony_system
python -m uvicorn workflow.api_server:app --host=0.0.0.0 --port=8000
```

看到类似下面内容，说明服务已启动：

```text
Application startup complete.
Uvicorn running on http://0.0.0.0:8000
```

启动成功后，这个窗口不要关闭。关闭窗口会停止服务。

本机检查服务：

```bat
curl http://127.0.0.1:8000/health
```

局域网其他电脑访问：

```text
http://新电脑IP:8000/health
```

如果提示端口 `8000` 已被占用，通常说明服务已经启动过。可以先检查：

```bat
netstat -ano | findstr :8000
```

如果确认要重启服务，先结束旧进程，再重新启动：

```bat
taskkill /PID 进程PID /F
```

也可以直接使用停止脚本 `D:\colony_system\stop_api.bat`，然后再执行启动命令。

如果只是现场日常使用，核心命令就是：

```bat
conda activate colony_system
cd /d D:\colony_system
python -m uvicorn workflow.api_server:app --host=0.0.0.0 --port=8000
```

## 1. 项目路径

新电脑项目根目录：

```text
D:\colony_system
```

由于当前项目中已有一些配置、任务模板和历史结果使用了旧路径：

```text
C:/colony_system
C:\colony_system
```

因此新电脑部署时，需要把仍然参与运行的配置文件同步为：

```text
D:/colony_system
D:\colony_system
```

重点检查：

```text
config/*.yaml
config/*.json
data/objective_state.json
start_api.bat
stop_api.bat
```

历史任务记录通常不用改：

```text
data/task_index/*.json
data/http_tests/*
data/vision_test_outputs/*
data/compensate_eval/*
```

这些多为旧运行结果，除非需要在新电脑上复查旧任务图片或旧结果。

## 2. 必须同步的路径项

### 2.1 物镜状态文件

文件：

```text
config/objectives.yaml
```

需要把：

```yaml
state:
  state_file: "C:/colony_system/data/objective_state.json"
```

改为：

```yaml
state:
  state_file: "D:/colony_system/data/objective_state.json"
```

同时确认文件存在：

```text
D:\colony_system\data\objective_state.json
```

### 2.2 任务模板输出路径

文件：

```text
config/task_*.json
```

重点字段：

```json
"save_dir": "D:/colony_system/data/..."
"output_json": "D:/colony_system/data/..."
"result_json": "D:/colony_system/data/..."
```

如果仍然是 `C:/colony_system/...`，任务可以启动，但输出会写到旧盘符路径，导致结果查询或文件下载异常。

### 2.3 自动对焦输出路径

文件：

```text
config/autofocus.yaml
third_party/XWJJJ260511/config.yaml
```

当前相对路径可以保留：

```yaml
output:
  timestamp_folder: true
  image_path: output/sharpest.png
  log_path: output/focus_log.csv
```

如果后续改成绝对路径，应统一使用：

```text
D:/colony_system/...
```

## 3. 服务 IP 和端口

服务启动命令：

```bat
python -m uvicorn workflow.api_server:app --host=0.0.0.0 --port=8000
```

说明：

```text
0.0.0.0 表示允许局域网访问，不是实际访问 IP。
8000 是服务端口。
```

在新电脑上查看实际 IP：

```bat
ipconfig
```

如果新电脑 IP 是：

```text
192.168.1.50
```

则其他电脑访问：

```text
http://192.168.1.50:8000/health
```

需要同步给上位机、前端或调用方：

```text
服务地址：http://新电脑IP:8000
健康检查：http://新电脑IP:8000/health
```

防火墙放行：

```bat
netsh advfirewall firewall add rule name="Colony System API 8000" dir=in action=allow protocol=TCP localport=8000
```

## 4. 相机 IP 和网卡 IP

文件：

```text
config/autofocus.yaml
third_party/XWJJJ260511/config.yaml
```

重点字段：

```yaml
camera:
  ip: 192.168.0.253
  net_export_ip: 192.168.0.10
```

同步规则：

```text
camera.ip            = 海康相机 IP
camera.net_export_ip = 新电脑连接相机的网卡 IP
```

如果相机 IP 保持：

```text
192.168.0.253
```

则新电脑相机网卡建议设置为同网段，例如：

```text
IP 地址：192.168.0.10
子网掩码：255.255.255.0
默认网关：留空
```

如果新电脑相机网卡改成 `192.168.0.20`，则两个配置文件里的 `net_export_ip` 都要同步改成：

```yaml
net_export_ip: 192.168.0.20
```

## 5. MVS SDK 路径

文件：

```text
config/autofocus.yaml
third_party/XWJJJ260511/config.yaml
devices/camera_controller.py
```

当前默认路径：

```text
C:/Program Files (x86)/MVS/Development/Samples/Python/MvImport
```

这个路径是海康 MVS SDK 的安装路径，通常不随项目盘符变化。

新电脑需要确认该路径存在。如果 MVS 安装到了其他位置，需要同步修改：

```yaml
mvs_sdk_path: 实际的 MvImport 路径
```

## 6. 串口 COM 口

文件：

```text
config/autofocus.yaml
config/handoff.yaml
config/objectives.yaml
config/task_*.json
```

当前常用配置：

```text
port: COM3
baudrate: 115200
```

新电脑需要在设备管理器中确认真实串口号。如果变成 `COM4`，需要把所有运行配置中的 `COM3` 同步为 `COM4`。

轴号也要确认：

```text
X 轴从站：1
Y 轴从站：2
调焦轴从站：3
物镜轴从站：4
```

## 7. 现场标定参数

以下文件应从旧电脑同步到新电脑，并在接硬件后复核：

```text
config/plates.yaml
config/handoff.yaml
config/objectives.yaml
data/objective_state.json
```

重点内容：

```text
plates.yaml       孔板 A1 起点、孔径、孔间距、脉冲/mm、安全限位
handoff.yaml      机械臂交接点坐标
objectives.yaml   4x/10x 物镜切换位置、调焦位置、轴号
objective_state   当前真实物镜状态
```

特别注意：

```text
data/objective_state.json 必须和真实物镜状态一致。
```

如果不确定真实物镜状态，不要直接跑自动切换或采集流程，先人工确认物镜位置。

## 8. Conda 环境

建议环境名保持一致：

```bat
conda create -n colony_system python=3.10 -y
conda activate colony_system
```

安装依赖：

```bat
pip install fastapi uvicorn pydantic pyyaml numpy opencv-python pillow pymodbus
```

检查依赖：

```bat
python -c "import fastapi, uvicorn, yaml, numpy, cv2, PIL, pymodbus; print('deps ok')"
```

## 9. 启动脚本

建议在新电脑创建：

```text
D:\colony_system\start_api.bat
```

内容：

```bat
@echo off
call C:\Users\你的用户名\anaconda3\Scripts\activate.bat colony_system
cd /d D:\colony_system
python -m uvicorn workflow.api_server:app --host=0.0.0.0 --port=8000
pause
```

如果不知道 Anaconda 安装路径，在 Anaconda Prompt 中执行：

```bat
where conda
```

根据实际路径修改 `activate.bat` 所在位置。

## 10. 停止脚本

建议创建：

```text
D:\colony_system\stop_api.bat
```

内容：

```bat
@echo off
for /f "tokens=5" %%a in ('netstat -ano ^| findstr :8000') do taskkill /PID %%a /F
pause
```

用于端口被占用或需要重启服务时：

```text
先执行 stop_api.bat，再执行 start_api.bat
```

## 11. 开机自启动

使用 Windows 任务计划程序：

```text
触发器：用户登录时，或开机时
操作：启动 D:\colony_system\start_api.bat
权限：使用最高权限运行
```

初期建议选择“用户登录时”，方便看到窗口和报错。稳定后再改为“开机时”。

## 12. 新电脑验收顺序

软件层：

```text
1. conda activate colony_system
2. cd /d D:\colony_system
3. python -m uvicorn workflow.api_server:app --host=0.0.0.0 --port=8000
4. curl http://127.0.0.1:8000/health
5. 其他电脑访问 http://新电脑IP:8000/health
```

硬件层：

```text
1. MVS 能识别相机
2. 相机网卡 IP 与 camera.net_export_ip 一致
3. 串口 COM 号正确
4. 相机拍照测试
5. handoff 点低速验证
6. 单孔 capture
7. pipeline capture + detect
8. compensate 补偿方向验证
```
